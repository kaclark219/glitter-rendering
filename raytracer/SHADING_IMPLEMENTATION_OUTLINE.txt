====================
4. ILLUMINATION MODEL
====================

4.1 Create Illumination Base Class (src/components/illumination.h)
   - Abstract base with pure virtual method:
     * Color illuminate(const IntersectData& data, const vector<LightSource*>& lights,
                       const Material& material, const Vec3& view_dir)

4.2 Phong Illumination Model (src/components/phong.h)
   - Implements illuminate() using Phong formula:
     * I = ka*Ia + Σ[kd*(N·L)*Il + ks*(R·V)^n*Il]
     * Where:
       - ka = ambient coefficient
       - Ia = ambient light intensity
       - kd = diffuse coefficient
       - N = surface normal (normalized)
       - L = light direction (normalized)
       - Il = light intensity/color
       - ks = specular coefficient
       - R = reflection of L around N
       - V = view direction (normalized)
       - n = shininess exponent

4.3 Blinn-Phong Alternative (Optional)
   - Uses halfway vector H = (L + V)/|L + V|
   - Specular: ks*(N·H)^n instead of (R·V)^n
   - Computationally cheaper, slightly different appearance

6.2 Ray Tracing Loop Modifications (render_cpu.cpp)
   - Instead of just finding nearest intersection:
     a) Find nearest intersection and store IntersectData
     b) If hit occurred:
        * Get hit point: origin + t * direction
        * Get surface normal: obj->normal(hit_point)
        * Get material from object's material index
        * Compute view direction: -ray.direction (normalized)
        * Call illumination model to get final color
        * Account for shadow rays (optional for basic shading)
     c) Set pixel to computed color

6.3 Shadow Ray Implementation (Optional but Recommended)
   - For each light source:
     * Cast shadow ray from hit_point to light position
     * Check if any object blocks the path (t between epsilon and 1.0)
     * If blocked, don't add that light's contribution
   - Prevents lighting from "passing through" objects

10. COMMON ISSUES TO AVOID
====================

- Surface acne: Use small epsilon when casting shadow rays
- Overly dark images: Check ambient light is sufficient
- Overly bright images: Clamp color values to 0-255
- Wrong normals: Ensure normals point outward from surface
- Integer overflow in Color: Use floats for computation, convert at end
- Non-normalized vectors: Normalize L, N, V before dot products
- Negative dot products: Clamp max(0, N·L) for diffuse/specular

====================
11. FORMULA REFERENCE
====================

Phong Shading Formula:
   I = Ia*ka + Σ[Il * (kd*(N·L) + ks*(R·V)^n)]

Where:
   Ia = ambient light intensity
   ka = ambient coefficient (0-1)
   Il = light intensity
   kd = diffuse coefficient (0-1)
   N = surface normal (unit vector)
   L = direction to light (unit vector)
   ks = specular coefficient (0-1)
   R = reflection of -L around N = 2*(N·L)*N - L
   V = view direction (unit vector, toward camera)
   n = shininess exponent (1-1000, typically 10-100)

Shadow Ray:
   shadow_origin = hit_point + epsilon * N
   shadow_dir = normalize(light_pos - hit_point)
   if any object intersects with t ∈ [epsilon, distance_to_light]:
      light is blocked, skip this light's contribution
